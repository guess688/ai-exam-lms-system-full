## 处理原则

1. 二次开发许可状态已更新为“用户确认已取得”，许可证不再作为当前阶段的阻塞项。
2. 进入 `external`、`backend` 或 `frontend` 的内容仍要记录来源仓库、文件路径、授权状态、迁移方式和修改范围。
3. 即使已取得授权，也不要把 Moodle、Frappe、LearnHouse、Studyield 的完整运行时强行混入主项目。
4. 直接复制代码只适合足够独立、依赖清晰、能被主项目维护的模块；其他内容优先抽设计后重写。
5. AI API Key、模型 Key、授权协议正文、私密令牌不得硬编码或提交到公开仓库。

## 入库和迁移时必须记录

后续若从 references 迁移任何内容到 `external`、`backend` 或 `frontend`，至少记录：

- 来源仓库和文件路径。
- 原始许可证和用户已取得的二次开发授权状态。
- 迁移方式：复制、改写、重写、接口兼容、独立服务。
- 修改内容和目标文件。
- 是否包含第三方依赖。
- 是否需要保留版权声明。
- 是否需要同步公开修改后的源码或按授权协议履行其他义务。

## 项目级建议

当前阶段建议采取“授权可用、架构克制”的策略。授权状态已更新，但正式业务代码仍应保持 SpringBoot + Vue 架构；能独立服务化的放入 `external`，与核心业务强耦合的部分在 `backend` 或 `frontend` 中按主项目风格重写。
